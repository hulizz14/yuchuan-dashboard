## 🧮 数据计算逻辑

### 1. 已售定义
```
已售 = 签约 + 认购
```
- **说明**：工抵是销售渠道属性，不是独立类别
- **数据筛选**：`销售状态 IN ('签约', '认购')`

### 2. 未售定义
```
未售 = 待售 + 销控
```
- **数据筛选**：`销售状态 IN ('待售', '销控')`

### 3. 关键指标计算

| 指标 | 计算公式 |
|-----|---------|
| 已售套数 | COUNT(销售状态 IN ('签约', '认购')) |
| 已售金额 | SUM(成交总价) WHERE 销售状态 IN ('签约', '认购') |
| 未售套数 | COUNT(销售状态 IN ('待售', '销控')) |
| 未售金额 | SUM(备案总价) WHERE 销售状态 IN ('待售', '销控') |
| 总货值 | 已售金额 + 未售金额 |
| 去化率 | (已售金额 / 总货值) × 100% |
| 已收款 | SUM(已收款) WHERE 销售状态 IN ('签约', '认购') |
| 未回款 | 已售金额 - 已收款 |
| 回款率 | (已收款 / 已售金额) × 100% |

### 4. 工抵统计逻辑
- 工抵是标记字段，不是状态字段
- **统计方式**：在已售/已签约/已认购数据中，统计`是否工抵 = '是'`的数量

## 🎨 看板设计说明

### 1. 页面结构
```
├─ Header（标题+核心KPI）
├─ 模块1：基础数据（分楼栋表格）
├─ 模块2：已售数据（CSS进度条图表）
├─ 模块3：未售数据（分楼栋卡片 - 重点）
└─ 模块4：已售未回款（汇总+明细）
```

### 2. 技术特点
- **纯HTML/CSS**：无外部图表库依赖，避免CDN加载失败
- **响应式布局**：使用Tailwind CSS网格系统
- **颜色标识**：
  - 绿色：去化率≥60%或已完成
  - 橙色：去化率30%-60%
  - 红色：去化率<30%或高风险

### 3. 重点关注
- **3号楼公寓**：未售318套，2.63亿，去化率仅11.7%，需重点跟进
